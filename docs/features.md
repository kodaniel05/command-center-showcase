# Features

## Project Registry

Command Center stores a structured registry of local projects. Each project can include a display name, description, local folder, launch command, port, health URL, project type, category, tags, framework, package manager, repository URL, preview metadata, and notes.

Public screenshot placeholder: `images/screenshots/add-project-modal.png`

## Project Dashboard

The dashboard presents registered projects as cards with preview images, metadata, status badges, and controls. It is designed for quick recognition and fast operations rather than deep navigation.

Public screenshot placeholder: `images/screenshots/dashboard-home.png`

## Runtime Controls

Each project can expose actions such as start, stop, restart, open in browser, open folder, open in editor, refresh metadata, view logs, capture a snapshot, and upload a preview image.

Public screenshot placeholder: `images/screenshots/project-card-runtime-controls.png`

## Health and Port Checks

Command Center validates startup conditions before and after launch. It checks whether folders exist, package scripts are available, ports are free, and local URLs become reachable.

Public screenshot placeholder: `images/screenshots/project-logs.png`

## Activity Feed

The activity feed records operational events such as starts, stops, restarts, errors, health-check results, browser opens, snapshot captures, uploads, replacements, and deletions.

Public screenshot placeholder: `images/screenshots/activity-feed.png`

## Snapshot Management

Project cards can show preview images. Screenshots may be captured automatically for running apps or uploaded manually when a project cannot be captured directly.

Public screenshot placeholder: `images/screenshots/snapshot-management.png`

## Notes

Project notes provide lightweight context such as debugging reminders, project status, or next-step notes. Notes should not contain secrets or sensitive client data.

Public screenshot placeholder: `images/screenshots/project-notes.png`

## System Overview

System overview cards show useful local machine signals such as memory usage, CPU information, active project count, and uptime.

Public screenshot placeholder: `images/screenshots/system-overview.png`

## Idea Blueprint Studio

Blueprint Studio converts a product idea into a structured technical plan. It can recommend a stack, estimate complexity, outline architecture, propose a file tree, list feature priorities, suggest API routes, and provide starter setup commands.

Public screenshot placeholder: `images/screenshots/idea-blueprint-studio.png`

## Blueprint Output

Generated blueprints are designed to be scannable and reusable. They help move from a vague idea to an implementation-ready plan while keeping scope visible.

Public screenshot placeholder: `images/screenshots/blueprint-output.png`

## Privacy-Aware Showcase Mode

The public repository documents the product without exposing private source code, secrets, exact private schemas, or cloneable implementation logic.
